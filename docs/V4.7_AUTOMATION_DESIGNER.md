# ShieldNet v4.7 Automation Designer

Automation Designer adds server-scoped no-code workflow definitions.

## Route

`/guild/{guild_id}/automations`

## Supported triggers

- member.joined
- member.left
- member.verified
- member.role_added
- moderation.case_created
- security.finding_created
- schedule.daily

## Supported actions

- member.add_role
- member.remove_role
- member.send_dm
- channel.send_message
- member.set_nickname
- audit.write
- webhook.call

Rules are stored as drafts and must pass validation before they can be enabled. v4.7 provides a dry-run evaluator and does not execute Discord changes. Live event execution will be connected to the Discord Worker in a later release after idempotency, rate-limit control and rollback are implemented.
