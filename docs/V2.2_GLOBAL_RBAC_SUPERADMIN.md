# ShieldNet v2.2 — Global RBAC & SuperAdmin Access

## Configuration

Add Discord user IDs to `/etc/shieldnet/backend/backend.env`:

```env
SHIELDNET_SUPERADMIN_IDS=123456789012345678,987654321098765432
```

Restart the backend:

```bash
systemctl restart shieldnet-backend
```

The user must sign in through Discord OAuth so that `discord_user_id` is linked to the ShieldNet account.

## API

- `GET /api/v1/auth/me` — includes effective roles, `highest_role`, and `is_superadmin`.
- `GET /api/v1/platform/access/me` — explains the active platform role and its source.
- `GET /api/v1/platform/access/overview` — SuperAdmin-only platform counters.

Environment-based access is evaluated on every request and does not require a database update.
