# ShieldNet v3.2 — Notification Center & Alert Rules

Adds a global SuperAdmin notification center at `/platform/notifications`.

Alert evaluation covers:

- stale backend, bot, and scheduler heartbeats;
- failed system jobs from the last 24 hours;
- overdue moderation cases grouped by Discord server;
- unresolved high and critical Security Center findings;
- unavailable Valkey/Redis queue service.

Alerts are deduplicated by fingerprint and support open, acknowledged, and resolved states. Acknowledgement, resolution, and manual rule evaluation are written to the Audit Log.

Database revision: `0027_alerts`.
