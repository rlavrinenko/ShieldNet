# ShieldNet v1.6 — Members Control Center

## Included

- Responsive Members Control Center at `/guild/:guildId/members`.
- Member statistics: humans, bots, activity, inactivity, pending, timeout and ShieldNet block totals.
- Search by username, nickname, global name, Discord ID and private admin note.
- Filters for member type and operational status; sorting by activity, name and join date.
- Paginated member list with roles, tags and activity information.
- Member inspector with Discord identity, dates, roles and current status.
- Private administrative notes and normalized member tags.
- Per-member Discord action history.
- Actions queued through the existing bot worker: DM, rename, role operations, ShieldNet block/unblock, kick and ban.
- Multi-select bulk DM and ShieldNet block/unblock operations (maximum UI page size: 50, API limit remains 200).
- PostgreSQL migration `0018_members_control_center`.

## Upgrade

```bash
cd /opt/shieldnet/backend
source venv/bin/activate
alembic upgrade head
sudo systemctl restart shieldnet-backend

cd /opt/shieldnet/admin-frontend
npm ci
npm run build
```

Publish `admin-frontend/dist/shieldnet-admin/browser/` using the existing Nginx deployment process.

## API additions

- `GET /api/v1/discord/guilds/{guild_id}/members/stats`
- `GET /api/v1/discord/guilds/{guild_id}/members/{discord_user_id}`
- `PATCH /api/v1/discord/guilds/{guild_id}/members/{discord_user_id}/profile`
- `GET /api/v1/discord/guilds/{guild_id}/members/{discord_user_id}/actions`
- Extended list filters on `GET /api/v1/discord/guilds/{guild_id}/members`
