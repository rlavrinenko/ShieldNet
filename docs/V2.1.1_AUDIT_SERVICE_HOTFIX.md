# ShieldNet v2.1.1 Audit Service Hotfix

Fixes `AttributeError: 'AuditService' object has no attribute 'record'` by
restoring a transaction-safe `AuditService.record()` method used by the
verification routes.

The method performs `flush()` only. The endpoint retains responsibility for
`commit()` or `rollback()`, so the business change and audit event remain in one
transaction.
