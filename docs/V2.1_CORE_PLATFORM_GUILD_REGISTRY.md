# ShieldNet v2.1 Core Platform — Guild Registry

v2.1 introduces one central registry for every Discord guild used by ShieldNet.

## Main changes

- OAuth now registers guilds owned by the authenticated Discord user.
- Owner memberships are created or reactivated automatically.
- Bot registration enriches the same guild record instead of creating a parallel state.
- Superadmin module access safely creates a temporary guild registry entry before any module writes a foreign key.
- Verification, Members, Moderation and future modules can rely on `discord.guilds` being present.
- New diagnostic endpoint: `GET /api/v1/discord/guild-registry/{guild_id}`.

Temporary records are updated with the real server name, icon, owner and member count after OAuth or bot synchronization.
