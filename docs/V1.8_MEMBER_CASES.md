# ShieldNet v1.8 — Member Cases & Incident Timeline

## Added

- Member-specific moderation and security cases.
- Categories: warning, spam, harassment, security, appeal and other.
- Severity: low, medium, high and critical.
- Workflow statuses: open, investigating, resolved and dismissed.
- Description, resolution, assigned administrator and audit timestamps.
- Case creation and status management directly from Members Control Center.
- PostgreSQL migration `0020_cases`.

## Upgrade

```bash
cd /opt/shieldnet/backend
source venv/bin/activate
alembic upgrade head
sudo systemctl restart shieldnet-backend

cd /opt/shieldnet/admin-frontend
npm ci
npm run build
```
