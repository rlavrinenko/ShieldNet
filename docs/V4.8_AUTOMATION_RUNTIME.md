# ShieldNet v4.8 Automation Runtime Engine

Enabled automation rules are now evaluated by the backend and executed by the Discord Worker.

## Runtime safety
- Per-event idempotency keys prevent duplicate runs.
- Action results are persisted in `system.automation_runs`.
- `stop_on_error` controls workflow continuation.
- A rule is automatically disabled after `max_failures` failed runs.
- Discord actions use the bot's normal permission and rate-limit handling.

## Live triggers
- `member.joined`
- `member.left`
- `member.role_added`

Other catalog triggers remain available for future producers.
