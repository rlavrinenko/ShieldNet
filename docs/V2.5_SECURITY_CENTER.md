# ShieldNet v2.5 — Security Center

## Added

- Security Center route: `/guild/<guild_id>/security`
- periodic Discord security snapshots from the bot every 15 minutes;
- analysis of roles with Administrator and broad management permissions;
- checks for excessive webhook management access;
- checks for unsafe `@everyone` channel permissions;
- webhook ownership and orphan checks;
- verification of ShieldNet bot audit/role permissions;
- risk score from 0 to 100;
- severity counters and actionable recommendations;
- PostgreSQL schema `security` with snapshots and findings.

## Migration

```text
0025_security
```

## Required service restart

Both services must be restarted because v2.5 changes the backend and Discord bot:

```bash
systemctl restart shieldnet-backend
systemctl restart shieldnet-bot
```

The first security snapshot normally appears within 15 minutes after the bot starts.
